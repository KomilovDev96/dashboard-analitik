# MRP Dashboard — Что сделано

## Стек технологий

**Бэкенд:** NestJS · TypeScript · MongoDB (Mongoose) · ClickHouse · Redis · JWT · SSE  
**Фронтенд:** React · TypeScript · Ant Design · Zustand · TanStack Query · Vite

---

## 1. Аутентификация

- Логин по email / паролю, JWT-токен (7 дней)
- Две роли: `super_admin` и `client`
- Защищённые роуты на фронтенде (`RequireAuth`, `RequireAdmin`)
- При неверном пароле — push-уведомление с текстом ошибки
- При успешном входе — приветственное уведомление с именем пользователя
- Кнопка «Войти» показывает спиннер во время запроса
- Автоматическое создание Super Admin при запуске сервера (SeedService)
- JWT-токен передаётся через `?token=` query-параметр для SSE (EventSource не поддерживает заголовки)

## 2. MRP Отчёт — Остатки на складах

### Фильтры
| Фильтр | Источник данных |
|--------|----------------|
| Дата (диапазон) | `ТоварыНаСкладах."Дата"` |
| Категория товара | `products_hierarchy."Уровень_1"` |
| Склад | `Склады."Наименование"` |

### SQL-запрос (связи по ТЗ)
```sql
SELECT
  n."Наименование"         AS product_name,
  ph."Уровень_1"           AS category,
  s."Наименование"         AS warehouse,
  SUM(t."КонечныйОстаток") AS balance
FROM "ТоварыНаСкладах" t
INNER JOIN "Номенклатура" n      ON t."Номенклатура_Key"    = n.uuid
INNER JOIN products_hierarchy ph ON n."ВидНоменклатуры_Key" = ph.uuid
INNER JOIN "Склады" s            ON t."Склад_Key"           = s.uuid
WHERE t."Дата" = (SELECT max("Дата") FROM "ТоварыНаСкладах" WHERE "Дата" <= '{dateTo}')
  AND ph."Уровень_1" IN (...)   -- фильтр категорий
  AND s."Наименование" IN (...)  -- фильтр складов
GROUP BY n."Наименование", ph."Уровень_1", s."Наименование"
ORDER BY ph."Уровень_1", n."Наименование"
```

### Логика баланса
`КонечныйОстаток` = остаток на конец дня. Суммировать за период нельзя.  
Решение: всегда берём остаток **на последнюю доступную дату ≤ dateTo**.

### Загрузка данных (SSE)
- Server-Sent Events — потоковая передача данных в реальном времени
- Прогресс-бар в шапке: `%` + `загружено X из Y записей`
- После загрузки — зелёная полоса «Загрузка завершена»
- Кнопка «Стоп» для отмены загрузки
- ClickHouse-клиент переписан на нативный `http`-модуль Node.js с `Buffer.from(sql, 'utf8')` — для корректной передачи кириллических названий таблиц

## 3. Таблица данных

- **Статистика:** Всего записей / После фильтра / Категорий / Общий остаток
- **Поиск** по номенклатуре, категории, складу (live-фильтрация)
- **Сортировка** по номенклатуре и остатку
- **Фильтрация** по категории и складу через колонки таблицы
- **Цветные теги** категорий (12-цветная палитра)
- **Цветные бейджи** остатка: 🔴 ≤ 0 / 🟠 < 10 / 🟢 ≥ 10
- **Пагинация** по 20 записей
- Скелетон-загрузка пока данных нет
- **Два пустых состояния:**
  - До загрузки: 📦 «Выберите фильтры и нажмите Загрузить»
  - После загрузки с 0 результатами: 🔍 «Данных нет — попробуйте изменить фильтры»

---

## 4. Управление пользователями (Super Admin)

- Список всех пользователей с ролью и статусом
- Создание нового клиента (имя, email, пароль, компания, телефон)
- Активация / деактивация аккаунта
- Удаление пользователя
- Доступно только для `super_admin`

---

## 5. Настройки профиля

- Аватар: загрузка фото (до 2MB), preview, хранение в MongoDB (base64)
- Поля: Имя, Фамилия, Телефон, Email, Должность (только super_admin)
- Изменение пароля с подтверждением текущего
- После сохранения аватар сразу отображается в шапке приложения
- Доступно для обеих ролей, поле Email редактируется только у super_admin

## 7. Кэширование (Redis)

| Ключ | TTL |
|------|-----|
| Список категорий | 10 мин |
| Список складов | 10 мин |
| Диапазон дат | 10 мин |
| Отчёт (по набору фильтров) | 2 мин |

При недоступном Redis — graceful fallback (запросы идут напрямую в ClickHouse).

---

## 8. Безопасность

- Пароли хешируются через `bcrypt` (10 раундов)
- `ValidationPipe` с `whitelist: true, forbidNonWhitelisted: true`
- `RolesGuard` — защита эндпоинтов по роли
- Axios-интерцептор: 401 → редирект на `/login` (кроме самого запроса логина)
- JWT-стратегия читает токен из заголовка `Authorization` или `?token=` query
