## Столбец 1 — Средние продажи в день за последние 6 месяцев

**Статус:** реализован в ветке `feature/avg-sales-column`

```sql
SELECT
  product_id,
  sum(quantity) / dateDiff('day',
    toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH),
    toDate(now())
  ) AS avg_daily_sales
FROM sales_analysis
WHERE sale_date >= toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH)
  AND quantity > 0
GROUP BY product_id
```

**JOIN в основном запросе:**
```sql
LEFT JOIN (...) avgs ON t."Номенклатура_Key" = avgs.product_id
```

## Столбец 3 — Проверочные столбцы (числитель и знаменатель)

**Статус:** реализовано 30.04.2026

Добавлены два вспомогательных столбца в таблицу MRP для проверки корректности расчёта `avg_daily_sales`.

**SQL (расширенный `AVG_SALES_SUBQUERY`):**
```sql
SELECT
  product_id,
  sum(quantity) AS total_sales_6m,
  dateDiff('day',
    toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH),
    toDate(now())
  ) AS days_count,
  sum(quantity) / dateDiff('day',
    toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH),
    toDate(now())
  ) AS avg_daily_sales
FROM sales_analysis
WHERE sale_date >= toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH)
  AND quantity > 0
GROUP BY product_id
```

**Новые столбцы в таблице:**

| Столбец | Поле | Описание |
|---|---|---|
| `Продажи 6 мес` | `total_sales_6m` | Числитель — сумма продаж за 6 мес. Зелёный бейдж. |
| `Кол-во дней` | `days_count` | Знаменатель — ~180 дней, одинаков для всех. Серый. |
| `Ср. продажи/день` | `avg_daily_sales` | Результат = `total_sales_6m / days_count`. Фиолетовый. |

Для строк-категорий (`level`) `total_sales_6m` суммируется по всем вложенным товарам.

**Изменённые файлы:**
- `server/src/mrp/mrp.service.ts` — расширен `AVG_SALES_SUBQUERY`, добавлены поля в `SELECT` и `MrpRow`
- `client/src/types/index.ts` — добавлены `total_sales_6m?`, `days_count?` в `MrpRow`
- `client/src/components/MrpTable/MrpDataTable.tsx` — обновлены интерфейсы `LevelRow`/`ProductRow`, `buildTree`, добавлены два столбца

## Важно

- **НЕ зависит от фильтра слева** — подзапрос всегда считает по всей таблице `sales_analysis`
- Период: с первого числа месяца, который был 6 месяцев назад, по сегодня



Колонка avg_daily_sales (ср. продажи/день)

Всегда берёт последние 6 месяцев с начала месяца, игнорирует фильтры:


WHERE sale_date >= toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH)
-- сейчас это ≈ 2025-11-01





Фильтр по дате влияет только на данные из ТоварыНаСкладах (остаток, в пути, заказано).

avg_daily_sales — полностью игнорирует любой фильтр, потому что это отдельный подзапрос, захардкоженный на now() - 6 месяцев:


-- Этот подзапрос всегда считает последние 6 месяцев
-- НЕ зависит от dateFrom/dateTo которые пришли в фильтре
WHERE sale_date >= toStartOfMonth(toDate(now()) - INTERVAL 6 MONTH)
Он джойнится к основному запросу только по product_id, поэтому:

Что выбрал пользователь	Остаток	В пути	Заказано	Ср. продажи/день
дата 01.01 → 29.04	по выбранному периоду	по выбранному	по выбранному	всегда последние 6 мес.
дата 01.03 → 29.04	по выбранному периоду	по выбранному	по выбранному	всегда последние 6 мес.
